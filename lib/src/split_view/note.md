拖动处理：
- 开始拖动时, 根据当前位置和拖动位置计算拖动距离
  - 记录当前分割线信息, 分割线左右面板可压缩的最小值、可扩展的最大值作为 clamp
  - 考虑容器最大尺寸, 分割线位置不超过容器位置
  - 取值区间：分割线左右第一项固定获取, 直到后方第一个固定尺寸的项为止
- 行为
  - 两侧均为固定项时, 一侧放大, 一侧缩小
  - 一侧固定, 一侧 flex 时, 固定侧独立放大缩小, flex 侧整体放大缩小
  - 两侧均为 flex 面板的项时, 更新 flex 的 px 尺寸, 并执行一次计算流程来更新 flex 比例
- 分割线位置更新的同时, 根据新的分割位置更新面板尺寸, 只需更新分割线相邻面板尺寸, 其他面板尺寸动态计算
- 需要拖动时在内存中就计算好面板尺寸：
  - 按行为处的说明更新相邻面板尺寸, 并应用 clamp
  - 更新分割线信息
  - 更新渲染 children
- 将新的尺寸应用到 UI, 内存中计算的尺寸不应该依赖于容器尺寸外的 UI 渲染结果

流程：
1. 接收初始化配置
2. 初次 build 时, 执行一次信息更新, 记录 infos、separators、children 等
3. 尺寸由于拖动等发生变更时, 更新 infos、separators、children, 如果任意一项有变化, 更新组件
- 容器尺寸更新时同样执行更新操作
